The Problem

Locked hands.
Critical information
out of reach.

During robotic surgery, the surgeon's hands are locked on instrument controls inside a sterile field. They cannot type, click, or interact with any computer system.

Critical patient data, imaging, and anatomical context require them to break scrub or rely on circulating staff — both slow, disruptive, and potentially dangerous at the wrong moment.

🔒
Sterile Field Constraint
Hands never leave the controls during critical phases
⏱
Information Latency
Relying on staff to retrieve data adds cognitive load and delay
🎯
Context Switching Cost
Breaking scrub for data interrupts surgical flow and focus
How It Works

From voice to display
in milliseconds

ORION runs a real-time multimodal pipeline: live audio in, native Gemini intelligence, specialist agents, and instant visual output.

🎙
Voice Command
Surgeon speaks. 16kHz audio streams to ORION via WebSocket in real time. Wake-word filtering ignores background OR noise.
🧠
Gemini Live API
Audio + live video (1fps) processed natively by Gemini 2.5 Flash. Intent understood in the full surgical context.
⚡
Multi-Agent Routing
ORION Orchestrator routes to the right specialist agent — patient data, CT imaging, 3D anatomy, or procedural context.
🖥
Visual Overlay
Results appear as glassmorphism tiles on the surgical display. ORION speaks the confirmation. Hands never move.
Specialist Agents

Five agents. One voice.

Each specialist is a dedicated Gemini agent with purpose-built tools and instructions. The orchestrator routes every command with zero latency.

🎛
ORION_Orchestrator
Root Agent — Wake-Word Detection & Routing

The always-on orchestrator receives all audio, filters ambient OR noise using wake-word detection, and routes each command to the correct specialist using ADK's AutoFlow. Handles hide-all overlays directly. Never says "routing" — just acts.

Wake-word filtering AutoFlow routing Multi-turn conversation Gemini 2.5 Flash Native Audio
📋
IR_Agent
Information Retrieval

Retrieves and displays patient clinical data on demand — lab values, vitals, medication lists, allergies, and diagnosis — as glassmorphism tiles.

Labs & vitals Medications Allergies
🔬
IV_Agent
CT Image Viewer

Navigates CT scan slices by voice. Jump to anatomical landmarks like carina, aortic arch, or tumor — or scroll slice-by-slice superior and inferior.

Slice navigation Landmark jump
🫁
AR_Agent
3D Anatomy Renderer

Controls a Three.js 3D lung anatomy model. Rotate to posterior, anterior, lateral or superior views. Toggle structure visibility by name — vessels, bronchi, tumor, pleura.

3D rotation Structure toggle
🧬
PC_Agent
Procedural Context — Live Video Phase Detection

The most advanced agent. Gemini already receives the live surgical video at 1fps via the realtime API. PC_Agent uses that visual context to detect the current surgical phase and surface a phase-specific 4-point anatomical checklist with critical warnings — no surgeon input needed to detect phase transitions.

Live video analysis Phase detection Anatomical checklists Critical warnings 8 surgical phases
